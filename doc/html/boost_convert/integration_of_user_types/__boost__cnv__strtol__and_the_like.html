boost::cnv::strtol and the Like

In other converters (all based on boost::cnv::cnvbase) support for user types has been implemented similarly but without the std::iostream-related overhead (see user-types-related part of the Converters Compared section). Namely, new types can be plugged in to the framework via: 

void operator>>(TypeIn const&, boost::optional<TypeOut>&);

For example, the mentioned change class is deployed with boost::cnv::strol after having the following defined: 

inline void operator>>(change const& chg, boost::optional<std::string>& str)
{
    str = chg.value() == change::up ? "up" : chg.value() == change::dn ? "dn" : "no";
}

inline void operator>>(std::string const& str, boost::optional<change>& chg)
{
    /**/ if (str == "up") chg = change::up;
    else if (str == "dn") chg = change::dn;
    else if (str == "no") chg = change::no;
}

That allows handling conversions of user-defined types with boost::cnv::strtol converter: 

boost::cnv::strtol cnv;
change          up_chg = change::up;
change          dn_chg = change::dn;

BOOST_TEST(convert<std::string>(up_chg, cnv, "bad") == "up");
BOOST_TEST(convert<std::string>(dn_chg, cnv, "bad") == "dn");
BOOST_TEST(convert<std::string>(    12, cnv, "bad") == "12");

BOOST_TEST(convert<change>("up", cnv, change::no) == change::up);
BOOST_TEST(convert<change>("dn", cnv, change::no) == change::dn);
BOOST_TEST(convert<   int>("12", cnv, -1) == 12);
